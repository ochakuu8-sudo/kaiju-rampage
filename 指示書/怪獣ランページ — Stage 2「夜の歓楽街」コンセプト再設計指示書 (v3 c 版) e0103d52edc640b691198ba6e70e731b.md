この指示書は **v3 (c 版)** です。各チャンクで「選択肢を 2-3 個に絞り、選ぶ判断基準を書く」形式にしてあります。実装 AI は最後の 1 つを選ぶだけで、候補を増やしたり、候補にないものを使うことは禁じます。

同時に **§4 機械化検出装置** (決定ログ・重複検出表・証拠形式チェック) で、機械的コピペが発生した瞬間に可視化します。

# 0. なぜ書き直すのか — 現状診断

現行 Stage 2 は、チャンクごとに建物の業種が違うだけで、**プレイヤーの体験としては全チャンクがほぼ同じ**になっています。原因は構造的なもので、建物を入れ替えるだけでは解消しません。

## 0.1 何がコンセプトを殺しているか

- **共通テンプレの末尾貼り付け**: 全チャンクに「上空 chouchin 帯」「facade ネオン両端」「歩道帯の街灯」「puddle_reflection」「路地猫」「電柱・電線・ジャンクションボックス」が同じ位置・同じ個数で挿入されている。これが視覚的均質化の最大要因。
- **HERO/AMBIENT メタの空洞化**: クラスタ宣言上は HERO が指定されているが、地面の広さ・光源の集中・道路形状・人の密度のどれもが HERO を支えておらず、宣言だけが浮いている。
- **道路レイアウトが固定**: ほぼ全チャンクが「中央 avenue + MID 横道路」のスケルトン。カメラに対する空間の体験が変わらない。
- **時刻の数字だけが進行する**: コメントには 24:00 → 26:00 → 28:00 → 04:30 と書かれているが、人数・光量・路面状態・店の開閉が時刻に追従していない。
- **対比が業種スワップで処理されている**: 「ラブホ街」「屋台」「映画館」の差別化が建物の種類だけで、空間の感触 (密/疎、明/暗、囲/開) が変わらない。
- **アンビエント 3 セルが独立した小芝居になっている**: HERO セルと意味的に繋がっておらず、画面が常に 4 つの個別シーンに分裂している。
- **床にも均質化テンプレが効いている**: Ch4–Ch8 の 5 チャンク連続で `asphalt` BASE 全面 + `red_carpet` 全幅 + `oil_stained_concrete` パッチ という同じ床骨格。Act II と Act III が床レベルで区別できていない。
- **Ch9–Ch10 に和風素材が誤って混入している**: `stone_pavement` / `fallen_leaves` / `moss` / `gravel` / `bamboo_cluster` / `kura` / `torii` 等が「神社予兆」として置かれているが、**Stage 3 は DOWNTOWN (都心オフィス街・朝) であり和風ではない**。旧設計 (Stage 3 = 神社) 前提の実装が残っている。

## 0.2 v1/v2/v3 の関係

- **v1 (具体例を書く)**: 実装 AI が具体例から**機械的にコピペ**して均質化
- **v2 初版 (方針のみ)**: 抽象方針だけだと AI が**自分でテンプレを作って**均質化 (= 「テンプレに値を埋める」モードに自動変換)
- **v3 = c 版 (本指示書)**: 各チャンクで**選択肢を 2-3 個に絞り、選ぶ判断基準を書く**。実装 AI は最後の選択 1 つだけする。同時に**機械化検出装置**を仕込む

## 0.3 直すべきレイヤー

建物リストではなく、**地面 / 光源 / 道路 / 人 / 時刻**の 5 レイヤーをチャンクごとに変動させる設計に切り替えます。建物は最後に決めます。

# 1. ステージ2 全体のコンセプト

<aside>
🌃

**Stage 2 は「歓楽街の一晩」を縦スクロールで体験する物語である。**

夕暮れの開店準備から始まり、深夜のピークを越え、明け方の静寂で次のステージへ手渡される。プレイヤーが下に進むにつれて時間が進む。

</aside>

## 1.1 一晩の弧 (4 幕)

Stage 2 の全チャンクは、以下の 4 幕のいずれかに必ず属します。チャンクは弧の上の「点」であり、独立した小ステージではありません。

- **Act I — 黄昏と仕込み**: 街がまだ完全には起きていない。光が点き始め、店主が暖簾を出し、客はまばら。
- **Act II — 点火**: 街が完全に「ON」になる瞬間。最も明るく、最も人が多く、最も賑やかな幕。
- **Act III — 過剰と影**: ピークが過ぎ、街が自分のテンションを持て余す。匿名性・性・倦怠・一人の影が前景化する。
- **Act IV — 余熱と朝**: 光が落ち、人が消え、路面の湿り気と猫だけが残る。次ステージ (都心オフィス街の朝) への引き渡し。

各幕は **3 チャンク**。Act 内で「導入 → 中心 → 受け渡し」の 3 拍を作ります。

## 1.2 弧を支えるグローバル変動軸

チャンク ID が進むごとに、以下の値が**単調に**変動します。Act の境界で折り返してはいけません。

- **人口密度**: Act II で最大、Act IV で最小。スパイクではなく山型。
- **光源の総量と種類**: Act I は点光源 (店先の提灯)、Act II は面発光 (ネオン壁・赤絨毯)、Act III は**局所光** (光が街全体に行き渡らず特定の場所だけ照らす — Ch6 は HERO 裏側に隠れた光・Ch7 は屋台周辺に集中する暖光・Ch8 は表のファサードと裏の暗がり)、Act IV は残光 (消し忘れ・自販機のみ)。
- **路面の状態**: 乾いた → 賑わいで汚れる → Act 内で乾と濡の対比が出る → 朝霧で再び乾く。Act III は単一の「濡」ではなく、**Ch ごとに乾と濡の局所対比**を作ること (Ch7 屋台周辺は乾、周辺は濡 等)。
- **音の想像 (BGM の前提)**: 賑わい → 喧騒 → くぐもり → 環境音のみ。
- **画角の閉塞感**: Act I は開けている、Act II〜III は囲まれる、Act IV で再び開く。

# 2. 設計原則 (全チャンク共通)

## 2.1 タグ先行の原則

- 1 チャンクごとに、**5 タグ (時刻 / 主役 / 人物層 / 主光源 / 主動線)** を埋めてから配置を始める。1 タグでも空白なら配置してはいけない。
- 各タグはファクトで書く (「夜街っぽい」「飲屋街感」のような抽象語・ジャンル連想に逃げない)。
    - ✅ 主役=「単独店舗 (ramen)」/ 主光源=「店内作業灯 + 畳まれた a_frame_sign」
    - ❌ 主役=「飲屋街」/ 主光源=「ネオン感」
- チャンク内の全要素は、5 タグのいずれかを**支えるか / 邪魔しないか**のいずれか。中立は不可。中立なものは置かない。

## 2.2 共通テンプレ全廃の原則

- 「夜街シグネチャ補強」「タイトパッキング埋め草」のような、**全チャンクに同じブロックを差し込む慣習を完全に廃止**する。
- Stage 全体で守るべき連続軸 (電柱・電線などの背景の縫い目) は、**チャンクごとに本数・位置・密度を変える**。Act IV では電線が切れていても良い。
- 「夜街っぽさ」は共通装飾で出すのではなく、**Act ごとの光源の質**で出す。

## 2.3 HERO 単一・スケール明示の原則

- 1 チャンクに HERO は **1 つだけ**。merged HERO を許すのは Act II Ch4 (交差点) と Act III Ch7 (屋台) の 2 回まで。
- HERO は以下の少なくとも **3 つ**で他を圧倒する。建物の種類だけで HERO 化してはいけない。
    - 占有面積 (床素材を専用色で広く敷く)
    - 光源の集中 (HERO セル光源数が他セル比 2 倍以上)
    - 道路の屈折 (HERO のためにだけ部分道路が走る、または横道路が消える)
    - 高さ (ファサード上空に固有の装飾帯を持つ)
- HERO 以外のセルは「HERO を見ている / 背を向けている / 通り過ぎている」のいずれかの態度を取る。**並置で終わらせない**。

## 2.4 アンビエントは目撃者の原則

- アンビエントセルは、HERO と関係のない別シーンであってはならない。**HERO に対する反応**として配置する。
- アンビエントの livingTrace (猫・自転車・ゴミ等) は、**HERO の存在を証言する向き**で置く。HERO の方を見る猫、HERO へ向かう自転車、HERO 由来のゴミ。

## 2.5 道路はメッセージの原則

- 横道路の有無・本数・部分幅は、そのチャンクの感情を決定する重要な変数。建物配置の後で決めてはいけない。**先に決める。**
- 「中央 avenue + MID 横道路」を Stage 2 のデフォルトにしない。**§3 の選択肢で許される範囲のみ**で組む。

## 2.6 時刻が見える原則

- 同じ建物でも、**営業中 / 仕込み中 / 閉店中 / 廃店中**で表情を変える。これは家具と人の配置で表現する (シャッター下りた看板、出されただけの暖簾、片付け中の椅子、蛍光灯だけ点いた無人の店)。
- 「人の数」だけで時刻を表現しない。**店の状態 × 路面 × 光**の三位一体で時刻を見せる。
- **API には「点灯/未点灯」「畳んだ/開いた」のような状態フラグはない**。状態は**家具の数・dy 位置・上空キャノピー有無・近接する小道具・人の有無**の組み合わせで暗示する。例: 「未点灯の `chouchin`」は API では表現できないので、`chouchin` を **×1–2 に減らし**、**上空キャノピーを置かず**、**店内に店主 1 人だけ** を置くことで「まだ点いていない店」を示す。「畳んだ `noren`」は API で表現できないので、`noren` を**店の入口より外側に置かず内側に置く**、または `noren` を**置かない**ことで示す。

## 2.7 隣接チャンク対比の原則

- 連続する 2 チャンクは、必ず以下のいずれかで明確に対比される。密 ↔ 疎、明 ↔ 暗、囲 ↔ 開、賑 ↔ 静、高 ↔ 低 (シルエットの天井)。
- 対比は「業種違い」ではない。**空間の感触の差**で取る。§4.4 弧チェックで「Ch_{N-1} と Ch_N で 5 軸のうち 3 つ以上が異なる」ことを検算する。

## 2.8 装飾の引き算原則

- 1 チャンクに使う家具の**種類数**に上限を設ける (**13 種以上は却下**)。種類が多いほど均質に見える。
- 「祭り感を出すために提灯・風船・旗・幟・パラソル・ベンチを全部置く」は **禁止**。Act II の点火チャンクですら、種類は絞って数で押す。
- 「夜街っぽいから puddle を置く」「飲屋街っぽいから猫を置く」のような**ジャンル連想による装飾**を禁ずる。装飾は §2.1 の 5 タグのいずれかを支える根拠が必要。

# 3. チャンクごとの設計ブリーフ (選択肢空間 + 判断基準)

各チャンクは以下の 5 ブロックで構成されます:

- **タグ** (時刻 / 主役 / 人物層 / 主光源 / 主動線)
- **5 軸**: 人 / 光 / 路面 / 音 / 画角
- **選択肢**: 各レイヤー (HERO / 床 BASE / 床 HERO アクセント / 道路 / 光源家具 / livingTrace 等) で**2-3 個**の候補
- **判断基準**: 候補のどれを選ぶかを決める基準
- **禁則**

実装 AI は候補から 1 つを選び、§4.1 決定ログにその選択理由を書きます。**候補を増やしたり、候補にないものを使ってはいけません**。逆に、候補をすべて使う必要もありません (HERO は 1 つだけ等)。

## Act I — 黄昏と仕込み (Ch0–Ch2)

### Ch0 — 街の入口、夕暮れの最後の客

- **タグ**: 時刻=18:30 / 主役=左右の床境界 (建物 HERO なし) / 人物層=家路の住人と最初の客 (少) / 主光源=店先の点光 (chouchin 0–2 未点灯) / 主動線=住宅側→歓楽街側の横断
- **5 軸**: 人=少 / 光=点 (まばらな店先) / 路面=乾 / 音=遠い喧騒 / 画角=開
- **選択肢**:
    - HERO の方向性: 「境界線」が HERO。**建物 HERO を置かない、人の流れも HERO ではない**。HERO は左右非対称な地面と光
    - 床 BASE: `residential_tile` または `concrete` (Stage 1 のパレットを引きずる)
    - 床 HERO アクセント: 歓楽街側のみ `wood_deck` または `asphalt` を細く敷く (面積小、左右非対称が目的)
    - 道路: 横道路 1 本 (全幅、住宅街の名残) + avenue 1 本 (中央 or 端、いずれか)
    - 光源家具: `chouchin` ×0–2 (未点灯) + `street_lamp` ×1–2 + `sign_board` ×1
    - livingTrace: `cat` ×0–1 / `bicycle_rack` ×0–1 / `mailbox` ×0–1 (合計 2 個まで)
- **判断基準**: NW/SW (左) と NE/SE (右) で**床素材を別物にする**ことが第一目的。建物 HERO は不要。光源は左右で密度が違うこと
- **禁則**: ネオンの面発光禁止 / 提灯のキャノピー禁止 / `puddle_reflection` 禁止 (まだ濡れていない) / `red_carpet` 禁止 (Act II 専売)

### Ch1 — 仕込みの店主

- **タグ**: 時刻=19:00 / 主役=営業前の単独店舗 (snack / izakaya / ramen / mahjong_parlor から 1) / 人物層=店主 1 人のみ / 主光源=店内作業灯 + 畳まれた a_frame_sign / 主動線=行き止まり感のある部分幅横道路 (片側切れ)
- **5 軸**: 人=極少 (店主のみ) / 光=作業灯 / 路面=乾 / 音=店内の物音 / 画角=やや閉
- **選択肢**:
    - HERO 建物 (1 つ): `snack` / `izakaya` / `ramen` / `mahjong_parlor` のいずれか (小型・営業前の単独店舗)
    - 床 BASE: `residential_tile` または `concrete`
    - 床 HERO アクセント: `wood_deck` (店先 1 セルのみ、面積小)
    - 道路: 横道路 1 本 (**部分幅**で「行き止まり感」を作る、左右どちらかが切れる)
    - 光源家具: `chouchin` ×1–2 (**未点灯を表現する小道具と組ませる**) + `a_frame_sign` ×1 (まだ畳んである) + `street_lamp` ×0–1
    - 店先小物: `noren` (出されていない or 裏返し) + `recycling_bin` または `milk_crate_stack`
    - 人: 店内に**1 人だけ** (店主)。他のセル 0
- **判断基準**: 「営業前」を**家具の状態**で示す (出されていない `noren`・未点灯 `chouchin`・空の客席)。HERO 建物の業種より、家具の状態のほうが先
- **禁則**: 客 (人) を店外に置かない / アンビエント 3 セルに賑わいを置かない / `puddle_reflection` 禁止 / 上空 `chouchin` キャノピー禁止

### Ch2 — 開店、最初の暖簾

- **タグ**: 時刻=19:30 / 主役=Ch1 と異なる業種の単独店 / 人物層=店主+最初の客 1 組 / 主光源=点灯 chouchin ×2–3 + 出された noren / 主動線=Ch1 と逆側が切れる部分幅横道路
- **5 軸**: 人=少 (店主+最初の客 1 組) / 光=点+一部面 / 路面=乾 / 音=暖簾の音 / 画角=開
- **選択肢**:
    - HERO 建物 (1 つ): Ch1 と**異なる業種**から選ぶ (`snack` / `izakaya` / `ramen` / `mahjong_parlor` のうち Ch1 と別のもの)
    - 床 BASE: `residential_tile` または `concrete` (Act I 共通)
    - 床 HERO アクセント: `wood_deck` (Ch1 と HERO セルの位置をずらす)
    - 道路: 横道路 1 本 (部分幅、Ch1 と切れる側を逆にする)
    - 光源家具: `chouchin` ×2–3 (**点灯**) + `a_frame_sign` ×1 (店から斜めに出された動的位置)
    - 店先小物: `noren` (店から半分はみ出した動的位置)
- **判断基準**: Ch1 (準備) と Ch2 (開店) の差は**家具の配置位置**で出す (家具種類差ではない)。`noren`・`chouchin` の状態が「動的」であること
- **禁則**: ピーク時の家具量を投入しない (Ch3 以降に取っておく) / 客を多数置かない / `puddle_reflection` 禁止

## Act II — 点火 (Ch3–Ch5)

### Ch3 — 大通りの賑わい

- **タグ**: 時刻=21:00 / 主役=両側ファサードのネオン壁 (建物 HERO なし) / 人物層=流れる群衆 (多) / 主光源=sign_board ×3–4 面発光 + 弱キャノピー chouchin / 主動線=中央でなく左右にずらした avenue
- **5 軸**: 人=多 / 光=面 (ネオン壁) / 路面=やや汚 / 音=喧騒の中心 / 画角=囲
- **選択肢**:
    - HERO の方向性: 「人の流れ」が HERO。**建物 HERO を中央に置かない**
    - 床 BASE: `red_carpet` または `tile` (光を返す床。**`asphalt` を主面にしない**)
    - 床 HERO アクセント: BASE と異なるほうを部分的に (BASE が `red_carpet` なら `tile`、逆も)
    - 道路: avenue 1 本 (**中央でなく左右にずらす**) + 横道路 0 本 or 部分幅 1 本
    - 光源家具: `chouchin` ×2–4 (キャノピーは弱め) + `sign_board` ×3–4 (両側面ファサード) + `banner_pole` ×1–2 + `vending` ×1
    - 建物アクセント (HERO 化しない): `snack` / `izakaya` / `karaoke` / `mahjong_parlor` を**両側に 3–5 棟並べる**
- **判断基準**: 「人の流れ」を**両側ファサードの sign_board 連続**で表現。HERO は建物ではなくプレイヤーを挟むネオン壁
- **禁則**: 中央に大型 HERO 建物を置かない / `puddle_reflection` 1 個まで / livingTrace 系を多用しない (流れが止まる) / `department_store` 禁止 (Ch4 専用)

### Ch4 — 交差点 (点火の頂点)

- **タグ**: 時刻=22:00 / 主役=4 セル横断の 1 行為 (信号待ち群衆 / 広告ジャック / 祭礼行列 から 1) / 人物層=最多群衆 / 主光源=上空 chouchin キャノピー ×4–6 + 4 方向 sign_board / 主動線=全幅十字 (avenue + 全幅横)
- **5 軸**: 人=最多 / 光=最大 / 路面=最も汚れる / 音=最大 / 画角=囲 (上下左右)
- **選択肢**:
    - merged HERO 許可枠 1/2
    - HERO 建物 (1 つ、merged の中心): `department_store` / `movie_theater` / `pachinko` のいずれか
    - 床 BASE: `red_carpet` または `tile` (Act II)

# 4. 機械化検出装置

§3 のブリーフを満たしただけでは均質化は防げません。実装後に**自分で機械化を検出する**仕組みを置きます。

## 4.1 決定ログ (各チャンクで必須)

各チャンクのコメントに、以下の **5 行**を**この順で**書く。書けないチャンクは設計が不足しているので進まない:

```jsx
// Ch_N 決定ログ
// 1. タグ (時刻=... / 主役=... / 人物層=... / 主光源=... / 主動線=...) — 前後チャンクとの差分でなぜこれか説明
// 2. 5 軸を選んだ理由: 人/光/路面/音/画角 のうち 2 つで具体に
// 3. 床 BASE を選んだ理由: §3 の候補のうちこれを選び、他を却下した理由

// 5. アンビエント 3 セルの態度: NW=見ている/背を向けている/通り過ぎている、NE=..., SW=..., SE=...
```

決定ログのないチャンクは**実装したと見なさない**。配置を先に書いて後から決定ログを埋めるのは**禁止** (これは機械化の偽装)。

- Ch9–Ch11 に `stone_pavement` / `fallen_leaves` / `moss` / `gravel` / `bamboo_cluster` / `torii` / `kura` / `shrine` を「神社予兆」として置くこと (Stage 3 は DOWNTOWN・現代オフィス街であり**和風ではない**)
- 配置を先に書いて後から決定ログを埋めること (機械化の偽装、§4.1 で禁止)

# 6. 実装の進め方 (順序の指定)

各チャンクを以下の順で:

1. §3 のブリーフを読む (タグ・5 軸・選択肢・判断基準・禁則)
2. **§4.1 決定ログの 5 行を先に書く** (書けないなら進まない)
3. 決定ログの内容に従って配置を生成 (選択肢から候補を選ぶだけ、候補外を使わない)
4. §4.3 の証拠形式チェックを通す (数値で答える)

12 チャンク全部実装したら:

1. §4.2 の重複検出表を作る
2. §4.4 の弧チェックを通す
3. 重複・矛盾があれば該当チャンクをやり直す

<aside>
📌

順序を破った瞬間に均質化が始まる。AI 側で順序を勝手に最適化しないこと。

特に「配置を先に書いて決定ログを後から埋める」は**機械化の偽装**として明確に禁止。

</aside>

# 7. 素材在庫 (ルックアップ表ではない)

<aside>
🧰

ここから先は **「使える素材の在庫」** だけです。**配置例も配役 (どの建物が何用) も書きません**。

§3 の選択肢で既にチャンクごとの候補は絞ってあります。ここはその選択肢に登場する素材の**寸法・カテゴリ**だけを示します。

</aside>

## 7.1 チャンクが返すべき構造

各チャンクは `RawChunkBody` を 1 つ返します:

- `buildings`: 建物の配置
- `furniture`: 家具の配置
- `humans`: 人の配置
- `grounds`: 床素材パッチ
- `horizontalRoads`: 横道路セグメント (部分幅可)
- `verticalRoads`: 縦道路セグメント (部分高さ可)
- `clusters` (オプション): セル単位の意味付け宣言 (`role: 'hero' | 'ambient'`、`cell: 'NW' | 'NE' | 'SW' | 'SE' | 'merged'`、`focal` / `companions` / `boundary` / `access` / `livingTrace` / `handoffTo`)

座標系: `dx`: ワールド X (`-180` 〜 `+180`) / `dy`: チャンクローカル Y (`0` 〜 `200`、`CHUNK_HEIGHT = 200`)。上空 (電線・電柱・facade 上段) は `dy ≥ 65` 帯。中央道路相当は `dy ≈ 100`。facade 下段は `dy ≈ 115〜125`。

## 7.2 ヘルパー API (シグネチャのみ)

`stages.ts` 冒頭に既にヘルパーが用意されています。**新しいヘルパーを発明しないこと**。

- `_B(size, dx, dy)` / `_F(type, dx, dy)` / `_H(dx, dy)`: 建物 / 家具 / 人を **clusters 参照なし**で追加
- `_G(type, dx, dy, w, h)`: 床素材パッチを追加
- `_HR(dy, xMin, xMax, cls?)`: 横道路 (`cls` は `'street'` か `'avenue'`、省略時 `'street'`)
- `_VR(dx, yMinLocal, yMaxLocal, cls?)`: 縦道路
- `$B(out, size, dx, dy)` → `BRef` / `$F(out, type, dx, dy)` → `FRef`: 配列に追加して**参照を返す**。`_CLUSTER` で参照する建物・家具はこの `$` 系で追加すること
- `_CLUSTER(out, { id, role, cell, focal, companions?, boundary?, access?, livingTrace?, handoffTo? })`

注意: `$B` / `$F` が返す `Ref` は配列途中への挿入で壊れます。**必ず末尾追加**。`_AVE` (中央 avenue) と `_MID_HR` (中央クロス横道路) は用意されていますが、**Stage 2 ではデフォルト採用しない**。§3 の選択肢に従って毎回設計し直す。

## 7.3 sidewalkZone

Stage 2 の `sidewalkZone` は **5 (夜街)** 固定。横道路沿いに歩道家具が自動配置されるので、`raw.furniture` で街路定番家具 (`vending` / `street_lamp` 等) を**中央道路沿いに重ね置きしない**。光源は HERO セルの**専用領域**に集中させる。

## 7.4 床素材 — Act 別の方針 (Ch 別指定はしない)

各 Act の BASE はこの方針に従って、**個別の Ch 選択は §3 の選択肢から実装 AI が選ぶ**。

- **Act I (Ch0–Ch2)**: BASE = `residential_tile` または `concrete` (Stage 1 のパレットを引きずる、`asphalt` を主面にしない)
- **Act II (Ch3–Ch5)**: BASE = `red_carpet` または `tile` (光を返す床)
- **Act III (Ch6–Ch8)**: BASE = `oil_stained_concrete` (光を吸う床)
- **Act IV (Ch9–Ch11)**: BASE = `concrete` または `tile` (彩度ゼロ、Stage 3 へ素材レベルで遷移)

Act 内 3 チャンクで BASE が同一でも可。差別化は §3 選択肢の他レイヤー (HERO アクセントの形・対比アクセントの登場有無・面積比) で。§4.2 重複検出で違反しないこと。

**使わない素材 (Stage 2 全体で禁止)**:

- `stone_pavement` / `fallen_leaves` / `moss` / `gravel` (Stage 3 = DOWNTOWN と矛盾、神社素材)
- `grass` / `dirt` (Stage 1 住宅街)
- `harbor_water` / `rust_deck` / `steel_plate` (Stage 4 港湾)

**限定使用 (§3 選択肢で明示された Ch でのみ)**:

- `hazard_stripe`: Ch9 で HERO が `parking` のときのみ
- `checker_tile`: Ch5 限定 (1 セル、面積小)
- `wood_deck`: Act I (Ch0–Ch2) のみ、店先小面積

## 7.7 アンチパターン (これをやったら没)

機械化の典型を**反例として明示**。AI はこれと一致したら却下:

❌ **Ch4 = department_store + hazard_stripe + chouchin ×4 + puddle ×2 + cat ×1**

→ 「Act II 点火 = 全部入りで派手」というジャンル連想。§3 Ch4 の「1 つの行為が 4 セルを横断」を支えていない。`hazard_stripe` は Ch9 (`parking`) 限定。

❌ **全チャンクで `asphalt` BASE + `red_carpet` 全幅 + `oil_stained_concrete` パッチ**

→ 旧 v1 床均質化の正体。§7.4 Act 別 BASE 切り替えに違反。

❌ **Ch9–Ch11 に `stone_pavement` / `fallen_leaves` / `moss` / `bamboo_cluster` / `torii` を「神社予兆」として配置**

→ 旧設計 (Stage 3 = 神社) の遺物。Stage 3 = DOWNTOWN と矛盾。

❌ **全チャンクの上空に `power_pole` ×4 + `power_line` ×2 + `cable_junction_box` ×2 が同じ位置**

→ 「夜街シグネチャ補強」の旧コピペ。§2.2 で禁止済み。

❌ **HERO 宣言だけして、地面・光源・道路で裏付けない**

→ §4.3 証拠形式チェックで 3 軸以上の優越が示せないなら HERO ではない。

❌ **「ジャンル連想」での装飾**

→ 「夜街だから puddle」「飲屋街だから cat」。§3 の判断基準にない理由で置いた家具は 1 つでも見つかれば却下。

❌ **配置を先に書いて、決定ログを後から埋める**

→ §4.1 で禁止。これは機械化の偽装。決定ログが先、配置が後。
