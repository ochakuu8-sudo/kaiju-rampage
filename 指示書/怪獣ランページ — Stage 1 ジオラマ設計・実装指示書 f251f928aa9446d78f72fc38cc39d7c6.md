# 怪獣ランページ — Stage 1 ジオラマ設計・実装指示書

<aside>
🏙️

このページは、`kaiju-rampage` の **Stage 1 を高品質な見下ろし型ジオラマへ再構成するための単体完結の設計・実装指示書**です。チャンクは実装上の分割単位であり、街並みの設計単位ではありません。道路の引き方、街区の分け方、建物・家具・人の帰属関係を含めて、1本の連続した住宅街として設計します。

</aside>

<aside>
🚨

**最重要**: この作業は「小物を増やす」ことではありません。既存または新規のチャンクを、**建物・家具・人・地面・道路が意味を持って配置されたジオラマ**へ再編成する作業です。すべてのオブジェクトに「ここにある理由」を持たせてください。

</aside>

<aside>
🧩

**この指示書の読み方**: 密度目安は「不足しているチャンクを完成水準へ引き上げるための基準」です。すでに密度を満たしているチャンクでは、さらに足すよりも、焦点・帰属・境界・動線・視認性を整理してください。方角ラベルやセル分割ではなく、道路沿い、店前、庭、公園外周、駅前、踏切前、ヤードなどの場所単位で設計します。

</aside>

<aside>
🎱

**ゲームプレイ前提**: ボールは破壊対象を貫通して進むため、広い通路を常に空ける必要はありません。ただし、線路・踏切・駅・広場・燃料補給源となる人流など、ゲーム上の読みに関わる焦点は家具で隠さないでください。重要なのは「物理的に通れる空間」よりも、ヒットしたい対象、壊したい場所、次へ進む方向が画面上で読めることです。

</aside>

## 0. 到達目標

Stage 1 を、以下の **住宅街から街はずれへ向かう都市縦断ストーリー**として成立させる。

```
Stage 1: 昼の住宅街
→ 小商店と公共広場
→ 学校・商店街・駅前
→ 田園・倉庫・踏切
```

完成後、画面だけを見て以下が分かること。

- 今いる場所がどんな街区か
- 各チャンクが何の場所か
- 人がなぜそこにいるか
- 家具がどの建物・道路・広場・生活痕跡に属しているか
- 地面パッチが場所の輪郭を作っているか
- 前後チャンクが自然につながっているか
- 破壊対象密度と燃料補給源が維持されているか

## 1. 実装前提

### 1.1 対象

```
Repository: https://github.com/ochakuu8-sudo/kaiju-rampage
Branch: main
Primary: src/stages.ts
Reference: src/constants.ts / src/entities.ts / src/scenes.ts / src/game.ts / src/humans.ts
```

### 1.2 raw チャンク形式

Stage 1 は、可能な限り `raw` チャンクとして手書き構成する。

```tsx
{
  patternId: 's1_raw',
  raw: {
    buildings: [
      _B('house', -120, 40),
    ],
    furniture: [
      _F('bench', 30, 125),
    ],
    humans: [
      _H(30, 125),
    ],
    grounds: [
      _G('concrete', 0, 100, 360, 200),
    ],
    horizontalRoads: [_MID_HR],
    verticalRoads: [..._SPINE_V],
  }
}
```

### 1.3 禁止プロパティ

以下を `src/stages.ts` に入れない。

```tsx
title: '...'
furnitures: [...]
roads: [...]
buildings: [...] // raw の外
furniture: [...] // raw の外
humans: [...]    // raw の外
grounds: [...]   // raw の外
```

### 1.4 道路設計の扱い

道路は、チャンクに付属する背景線ではなく、**街並み全体の骨格**として設計する。

現状コードでは、raw チャンクの道路は `horizontalRoads` と `verticalRoads` で直接定義される。Stage 1 では、基本的に以下のスパインが使われている。

```tsx
// 中央横道路: チャンク中央を左右に通す
const _MID_HR = _HR(100, -180, 180);

// 上端横道路: チャンク境界や大きな交差点で使う
const _TOP_HR = _HR(200, -180, 180);

// 縦スパイン: 左路地、中央大通り、右路地
const _SPINE_V = [
  _VR(-90, 0, 200),
  _VR(0, 0, 200, 'avenue'),
  _VR(+90, 0, 200),
];
```

ただし、この形を機械的に固定しなくてよい。重要なのは、**道路がチャンク境界をまたいで自然につながり、街の用途と人流を説明していること**である。

#### 道路設計の基本方針

- チャンクは保存単位であり、街区・道路・広場の単位ではない。
- 道路はチャンクごとに完結させず、前後3〜5チャンク単位で連続する導線として設計する。
- `x=0` の中央 avenue は、使う場合は Stage 1 全体の主導線として扱う。
- `x=-90` と `x=+90` の縦 street は、脇道、生活路地、裏通り、搬入路として扱う。
- `dy=100` の中央横道路は、チャンク内部を上下に割る線ではなく、店前・公共施設前・踏切前・広場前などの「横断導線」として意味を持たせる。
- `_TOP_HR` は単なる追加道路ではなく、次チャンクとの接続、交差点、駅前、公共広場のために使う。
- 道路を増やす場合は、建物と家具の配置も道路に合わせて再設計する。道路だけ増やして建物をそのままにしない。

### 1.5 型安全

実装前に `BuildingSize` / `FurnitureType` / `GroundType` を確認する。存在しない型は、意味が近い実在型へ置換する。

- 木造住宅: `kominka` / `bungalow` / `house`
- 現代住宅: `townhouse` / `mansion` / `house`
- 銭湯: `onsen_inn` + `bathhouse_chimney` + `noren`
- 商店街: `ramen` / `izakaya` / `bookstore` / `cafe` / `convenience`
- 学校・公共施設: `school` / `clinic` / `post_office` / `city_hall` / `library`
- 駅前: `train_station` + `platform_edge` + `bus_stop` + `taxi_rank_sign`
- 街はずれ: `farmhouse` / `greenhouse` / `barn` / `warehouse` / `garage` / `shed`

### 1.6 `fallen_leaves`

`fallen_leaves` は `GroundType`。家具として置かない。

```tsx
// NG
_F('fallen_leaves', 0, 100)

// OK
_G('fallen_leaves', 0, 100, 40, 20)
```

## 2. Stage 1 共通ジオラマ設計ルール

### 2.1 設計単位はチャンクではなく「道路・街区・広場」

この指示書では、チャンクを街並みの設計単位として扱わない。チャンクは `src/stages.ts` にデータを入れるための分割単位であり、完成画面では境界が見えない前提で設計する。

設計時は、まず以下を決める。

1. **主道路**: Stage 1 を貫く一番強い生活導線
2. **副道路**: 生活路地、裏通り、農道、搬入路
3. **交差点・広場**: 人が滞留する焦点
4. **街区**: 道路に面して建物が並ぶまとまり
5. **境界**: 柵、植栽、ガードレール、地面素材で区画を切る線

その後、必要に応じて実装上のチャンクへ分割する。したがって、1つの広場や駅前ロータリーが2チャンクにまたがってもよい。逆に、1チャンク内に無理に4つの見せ場を作る必要はない。

### 2.2 道路から街並みを組み立てる順序

実装AIは、Stage 1 の各区間を以下の順で作る。

1. **道路を引く**
    - 主道路、副道路、横断道路、上端接続道路を決める。
    - チャンク境界で道路が急に切れないよう、前後チャンクの `horizontalRoads` / `verticalRoads` を確認する。
2. **街区を切る**
    - 道路に面する建物列、裏側、庭、畑、ヤード、広場、駅前を分ける。
    - 建物を置けない狭い余白は、歩道・植栽・電柱・看板・ガードレールなどにする。
3. **焦点を置く**
    - 公園、銭湯、駅、学校、踏切など、1区間に1つの主役を置く。
    - 主役は道路や広場から見える位置に置く。道路から孤立させない。
4. **建物の向きを道路へ向ける**
    - 店舗、公共施設、駅は道路や広場にファサードを向ける。
    - 住宅は道路に面した玄関、庭、郵便受け、物干しで生活感を作る。
5. **家具を帰属させる**
    - 家具は「どの建物・道路・広場・庭・搬入口に属するか」を決めて置く。
    - 家具を密度埋めとしてランダム散布しない。
6. **人流を置く**
    - 人は入口、待機列、横断地点、ベンチ、店前、作業場、農道に置く。
    - 人の位置で、その場所の使われ方が分かるようにする。

### 2.3 セル単位設計を禁止する

1チャンクは 360px × 200px しかない。ここに道路、歩道、建物、庭、店先、家具、人を入れるため、`NW / NE / SW / SE` のような **2×2セルを設計単位にしてはいけない**。

セル単位で設計すると、実装AIが「4つの小箱をすべて埋める」方向に解釈しやすい。その結果、以下の問題が起きる。

- 1セル内に要素を詰め込みすぎて窮屈になる
- 逆に各セルが薄くなり、全体がスカスカになる
- 道路と街区のつながりより、セル埋めが優先される
- 駅前、広場、学校、公園、踏切のような面の設計が小さく分断される

標準構成は以下とする。

- 主役エリア: 1つ
- 補助エリア: 1〜2つ
- 残り: 道路、歩道、庭、余白、次区間への接続

設計単位はセルではなく、以下にする。

- 道路沿い
- 店前
- 住宅前
- 庭
- 公園外周
- 広場
- 駅前ロータリー
- 踏切前
- 裏路地
- 農道
- 搬入路
- ヤード

高密度に見せる場合も、建物を無理に増やすのではなく、道路沿いの店先、庭先、歩道、裏口、広場外周に家具クラスターを作って密度を上げる。

### 2.4 4層構造

主役エリアは必ず以下で作る。

1. **焦点**: 一目で場所を表す主役
2. **取り巻き**: 焦点の意味を補強する家具
3. **境界**: 地面・柵・植栽・ガードレールで輪郭を作る
4. **動線**: 人の入口・待機・通行・滞留を示す

### 2.5 密度の作り方

- 建物は意味を作る。空白埋めに使わない。
- 家具は密度を作る。ただし必ず帰属先を持たせる。
- 地面は輪郭と素材感を作る。
- 人は活動を示す。ランダム散布しない。
- 道路は街区の境界であり、人流の理由であり、次エリアへの接続である。

## 3. Stage 1 密度目安

以下は **不足しているチャンクを完成水準へ引き上げるための目安** である。すでにこの範囲に達している場合は、数を増やさず、焦点の読みやすさ、家具の帰属、地面境界、人流の整理を優先する。

- 通常チャンク
    - 建物: 16〜22
    - 家具: 55〜80
    - 人: 6〜12
    - 地面パッチ: 5〜9
- 高密度チャンク
    - 対象: S1-Ch3、S1-Ch5、S1-Ch6、S1-Ch7、S1-Ch8
    - 建物: 20〜28
    - 家具: 75〜100
    - 人: 10〜18
    - 地面パッチ: 7〜12
- 低密度に見せる必要があるチャンク
    - 対象: S1-Ch9、S1-Ch10、S1-Ch11
    - 建物: 14〜22
    - 家具: 55〜85
    - 人: 5〜12
    - 地面パッチ: 6〜10
    - 注意: 田園・倉庫・踏切は空白が意味になるが、画面が寂しくならないよう、柵・電柱・道路脇・積荷・草地・水たまりで密度を作る

### Stage 1 の増補・整理ルール

- 追加は不足している場所に対して行う。すでに完成密度を満たしている場所では、同種家具の重複を整理し、主役エリアの焦点・取り巻き・境界・動線が読みやすい状態へ整える。
- 住宅街チャンクでは、各住宅に `mailbox`、`bicycle`、`laundry_pole`、`potted_plant`、`ac_unit`、`hedge`、`flower_bed`、`cat` のうち2〜4個を帰属させる。
- 商店チャンクでは、各店舗に `shop_awning`、`noren`、`a_frame_sign`、`vending`、`bicycle_rack`、`bench`、`flower_planter_row` のうち2〜5個を帰属させる。
- 公園・広場では、遊具や焦点物だけで止めず、`bench`、`flower_bed`、`hedge`、`tree`、`bush`、`street_lamp`、`garbage`、`bicycle` を外周に追加する。
- 公共施設では、`flag_pole`、`post_box`、`atm`、`bench`、`bicycle_rack`、`street_lamp`、`bollard`、`plaza_tile_circle` を施設前に配置する。
- 道路沿いでは、`traffic_light`、`guardrail_short`、`street_lamp`、`manhole_cover`、`power_pole`、`power_line`、`street_mirror` を連続要素として配置する。
- 裏側や生活痕跡として、`garbage`、`recycling_bin`、`dumpster`、`ac_outdoor_cluster`、`cable_junction_box`、`milk_crate_stack` を建物裏や路地に追加する。
- 地面は `residential_tile` 1枚で終わらせず、庭、店先、歩道、公園、駐車場、踏切、倉庫前を `grass`、`dirt`、`concrete`、`tile`、`asphalt`、`wood_deck`、`fallen_leaves` で分ける。

# Stage 1 — 昼の住宅街から街はずれへ

## 目的

生活のある昼の住宅街を基準値として見せる。住宅、公園、公共施設、小商店、学校、商店街、駅、田園、倉庫、踏切を通って、街はずれまで自然につなぐ。

## 連続軸

- Ch0〜Ch5: `sakura_tree`
- Ch6〜Ch8: `tree`
- Ch9〜Ch11: `pine_tree`
- 全体: 電柱・電線、生活歩道、猫
- 地面: `residential_tile` → `concrete` → `grass` / `dirt` / `asphalt`

## Stage 1 道路主導の詳細設計（自然言語版）

<aside>
🧭

ここから先はコード例ではなく、**コーディングAIに渡すための自然言語の詳細設計**です。実装時はこの設計を読み取り、`src/stages.ts` の実在型に合わせて `raw` 配列へ変換してください。チャンク単位で無理に完結させず、道路・街区・広場・人流が前後につながるように実装します。`NW / NE / SW / SE` のようなセル単位では設計しません。

</aside>

### Stage 1 の道路骨格

Stage 1 は、住宅街から街はずれへ向かう生活道路として設計する。現状の `x=-90 / x=0 / x=+90` の縦スパインと `dy=100` の横道路は、単なる罫線ではなく、以下の意味を持たせる。

- `x=0` の中央 avenue
    - 住宅街を南北に貫く生活通り。
    - 店、公共施設、学校、駅、踏切へつながる一番分かりやすい導線。
    - ただし大都会の幹線道路にしない。生活道路として、街灯、マンホール、短いガードレール、横断地点、人の滞留で見せる。
- `x=-90` の左 street
    - 住宅の前道、庭先、商店街の裏側、農道、倉庫脇の搬入路として使う。
    - 密度は建物より、郵便受け、植木、物干し、自転車、電柱、電線で作る。
- `x=+90` の右 street
    - 店舗の脇道、公共施設の入口側、駅前のロータリー脇、踏切への接続路として使う。
    - 後半に近づくほど、濡れ路面、踏切、街はずれ感を少しずつ増やす。
- `dy=100` の中央横道路
    - 1チャンクを上下に切る線ではなく、区間ごとの横断導線。
    - 公園前、公共施設前、商店街の通り、学校前、駅前、踏切前として意味を変える。
- `_TOP_HR`
    - Ch2、Ch6、Ch10、Ch11 など、公共性が高い区間や次区間への接続が必要な場所で使う。
    - 上端に道路を足す場合は、次チャンク下端との接続として、信号、街灯、ガードレール、地面素材の連続を作る。

### Stage 1 の区間設計

- Ch0〜Ch2: 住宅街の生活道路
    - 中央 avenue は穏やかな生活通り。
    - 横道路は公園前、公共施設前の横断導線として使う。
    - 建物は道路に対して玄関・庭・郵便受けが向くように置く。
- Ch3〜Ch5: 小商店と公共広場
    - 中央 avenue と横道路の交差を、店先テラスや civic plaza の人流へ変える。
    - 店舗ファサードは道路に面する。家具は必ず店先に帰属させる。
- Ch6〜Ch8: 学校、商店街、駅
    - 道路は通学路、商店街の歩行帯、駅前ロータリーへ意味を変える。
    - Ch8 では道路・駅・線路・バス停・タクシー乗り場が接続するように作る。
- Ch9〜Ch11: 街はずれ、倉庫、踏切
    - 空白を畑、農道、倉庫前ヤード、踏切待ちスペースとして意味化する。
    - Ch11 は踏切を優先し、道路と線路の交差関係が一目で分かるようにする。

### S1-Ch0: 日本庭園と縁側のある住宅街入口

- **コンセプト**: Stage 1 の入口として、古い家の庭と現代住宅が混在する静かな住宅街を見せる。最初のチャンクなので、密度よりも「庭・住宅・生活道路」の読みやすさを優先する。
- **主役エリア**: 古民家の庭を主役にする。池、飛び石、盆栽、石灯籠、松、落ち葉をひとまとまりにして、小さな日本庭園として読めるようにする。庭園内に建物を詰め込まず、芝・土・落ち葉で余白に意味を持たせる。
- **周辺住宅**: 庭園の周囲に現代住宅や小さな住宅を置き、古い家と新しい家が混在する住宅街にする。各住宅の前には郵便受け、植木鉢、自転車、室外機、低い生垣、物干しなどを帰属させる。
- **道路と生活痕跡**: 中央 avenue と左右の生活路地には、街灯、マンホール、電柱、電線、植え込みを置く。道路脇の空白は、住宅の庭、玄関前、路地の余白として処理し、何もないタイル面を残さない。
- **人流**: 庭を眺める住人、縁側付近の住人、道路へ出る人を少数置く。人の数は控えめにし、静かな住宅街の導入として読ませる。
- **完成密度**: 建物16〜20、家具55〜70、人6〜10、地面パッチ6〜9を目安にする。
- **失敗条件**: 庭園だけ作って周辺住宅が薄い。庭の小物がランダムに散っていて庭園として読めない。道路脇が空白のまま。

### S1-Ch1: 児童公園を中心にした住宅街

- **コンセプト**: 住宅街の中にある小公園を主役にし、公園を利用する人と周囲の生活が一緒に見えるチャンクにする。公園単体ではなく、住宅街の中の公園として成立させる。
- **主役エリア**: 児童公園を、滑り台、ブランコ、砂場、遊具、ベンチ、花壇、生垣、ゴミ箱まで含む小さな面として作る。遊具だけを置かず、公園外周と入口を必ず作る。
- **周辺住宅と生活店舗**: 公園の周囲には住宅列と小さな生活店舗を置く。住宅には郵便受け、自転車、物干し、植木鉢、猫を帰属させる。店舗には自販機、看板、店先ベンチ、自転車ラックを置き、公園利用者が立ち寄る場所として見せる。
- **道路と境界**: 公園前の横道路は、安全な横断導線として扱う。電柱、電線、街灯、マンホール、低いガードレールを道路沿いに置き、公園と住宅の境界を生垣や地面パッチで整理する。
- **人流**: 遊具周りの子ども、ベンチにいる大人、公園入口の人、店舗へ向かう人を分けて配置する。
- **完成密度**: 建物16〜22、家具60〜85、人8〜14、地面パッチ6〜10を目安にする。
- **失敗条件**: 遊具が数個あるだけで公園の外周がない。公園と住宅・店がつながっていない。人が遊具の周りにいない。

### S1-Ch2: 保育園・診療所・郵便局が並ぶ生活公共区画

- **コンセプト**: 住宅街の中にある小さな公共区画として、保育園、診療所、郵便局をそれぞれ別用途として読ませる。歩道橋と道路小物で、生活道路より少し交通量のある場所にする。
- **主役エリア**: 保育園、診療所、郵便局を道路に面した公共施設列としてまとめる。保育園には砂場、小さな遊具、旗、低いフェンス、送迎待ちの人を置く。診療所前にはベンチと待ち人、郵便局前にはポスト、旗、自転車列を置く。
- **道路と歩道橋**: 歩道橋、信号、短いガードレール、街灯、マンホールを公共施設前の道路に帰属させる。歩道橋は単体で浮かせず、横断地点、施設入口、人の待機位置とセットで見せる。
- **向かい側の生活感**: 公共区画の周囲には住宅を少し残し、郵便受け、植木、物干し、室外機で落ち着いた生活感を作る。
- **人流**: 保育園の送迎、診療所の待ち人、郵便局利用者、横断地点や歩道橋付近の通行人を分けて配置する。
- **完成密度**: 建物18〜24、家具65〜90、人8〜14、地面パッチ7〜10を目安にする。
- **失敗条件**: 3施設がただ並ぶだけで用途差がない。歩道橋が単体で浮いている。施設前に人がいない。

### S1-Ch3: 3連カフェテラスと小商店の並び

- **コンセプト**: 住宅街から商店街へ密度が上がる最初のチャンク。ベーカリー、本屋、カフェを中心に、店先テラスと歩行者で「歩いて楽しい昼の通り」を作る。
- **主役エリア**: ベーカリー、本屋、カフェの店先を連続したテラスとして作る。オーニング、看板、パラソル、ベンチ、花壇、自転車列を各店に帰属させ、3店舗が横並びの商店街として一目で分かるようにする。
- **周辺の奥行き**: 生活店舗、ランドリー、薬局、小住宅を混ぜ、住宅街から商店街へ変化している途中の奥行きを作る。建物は一直線に並べすぎず、道路に面したファサードが読める範囲で少しずらす。
- **道路と店前導線**: 中央 avenue と横道路の交差は、車道ではなく店前を歩く人の導線として扱う。地面は店先デッキ、歩道、道路で素材を分ける。
- **人流**: 店の入口、テラス席、通行人を分けて配置する。人は店前に集め、住宅街から商店街へにぎわいが増えたことを示す。
- **完成密度**: 建物20〜28、家具75〜100、人10〜18、地面パッチ7〜12を目安にする。
- **失敗条件**: 店が3つあるだけで店先のにぎわいがない。パラソルやベンチが少なく、テラスに見えない。人流が店前に集中していない。

### S1-Ch4: 銭湯風施設と生活商店の交差点

- **コンセプト**: 住宅街の中の銭湯と生活商店が交差点に集まる場所にする。`onsen_inn` 系の建物をそのまま置くだけでは銭湯に見えないため、煙突、暖簾、牛乳ケース、ベンチ、自販機で意味を補強する。
- **主役エリア**: 銭湯風施設を主役にし、煙突、暖簾、牛乳ケース、ベンチ、自販機、湯上がりの人を入口前にまとめる。地面はタイルやコンクリートで銭湯入口を切り出す。
- **生活商店と飲食店**: スーパー、薬局、ラーメン店などを道路に面して置く。店先にはオーニング、自転車ラック、看板、暖簾、待ち人を帰属させる。夜街のようにせず、昼の生活商店として読む。
- **住宅と裏側**: 住宅や集合住宅を少し残し、洗濯物、ベランダ、室外機、猫、ゴミ出し場、裏口小物で生活感を作る。
- **人流**: 銭湯入口の人、買い物客、ラーメン店の待ち人、住宅前の住人を分けて配置する。
- **完成密度**: 建物18〜24、家具65〜90、人8〜14、地面パッチ6〜10を目安にする。
- **失敗条件**: 銭湯風施設が旅館に見える。店と住宅の小物が混ざって帰属先が分からない。裏側の生活痕跡がない。

### S1-Ch5: 小さな公共広場と civic plaza

- **コンセプト**: Stage 1 前半の公共中心地として、郵便局、銀行、市役所、図書館、カフェが小広場を囲む civic plaza にする。整然とした公共性と日常利用の休憩感を両立する。
- **主役エリア**: 広場を、円形タイル、像、噴水、花壇、ベンチで作る。広場中央は見通しを残し、家具で埋めすぎず、外周に人が滞留する構図にする。
- **公共施設の用途差**: 郵便局にはポストと待ち人、銀行にはATMと旗、市役所や図書館には旗、ベンチ、街灯、入口前の人を帰属させる。施設名がなくても用途差が小物で読めるようにする。
- **カフェと日常利用**: 広場の一角にカフェや休憩要素を置き、公共施設だけでなく日常的に使われる場所として見せる。
- **人流**: 施設入口の人、広場で休む人、噴水や像の近くにいる人、カフェ利用者を分けて配置する。
- **完成密度**: 建物20〜28、家具75〜100、人10〜18、地面パッチ8〜12を目安にする。
- **失敗条件**: 広場中央に建物や大量家具を置いて焦点を隠す。公共施設の用途差が小物で表現されていない。人が広場に滞留していない。

### S1-Ch6: 小学校キャンパスと通学路

- **コンセプト**: 学校建物だけでなく、校庭、遊具、通学路、横断地点まで含めた小学校キャンパスとして作る。Stage 1 の中でも高密度にしてよいが、学校前としての読みやすさを優先する。
- **主役エリア**: 小学校校舎、校門、旗、像、植栽をまとめ、学校入口として一目で読めるようにする。校舎だけを置かず、入口と校庭の関係を作る。
- **校庭**: 砂地を大きく取り、滑り台、ブランコ、砂場、ジャングルジム、ベンチを外周まで配置する。遊具はランダムに置かず、校庭の中で遊び場としてまとまるようにする。
- **通学路**: 学校前の道路には、信号、ガードレール、街灯、マンホール、横断地点を置く。道路は学校に向かう通学路として機能させる。
- **周辺施設**: 保育園や周辺店舗を少し置き、学校周辺の生活施設として見せる。店先には看板、自転車、待ち人を配置する。
- **人流**: 校門前の人、校庭で遊ぶ人、見守り役、横断地点の通行人、店前の人を分けて配置する。
- **完成密度**: 建物20〜28、家具80〜100、人12〜18、地面パッチ8〜12を目安にする。
- **失敗条件**: 学校建物だけで終わる。校庭が空白。遊具と人の対応がない。道路が学校前として読めない。

### S1-Ch7: 昼の提灯アーケード商店街

- **コンセプト**: 昼のローカル商店街として成立させる。提灯や暖簾は使うが、夜の歓楽街になりすぎないよう、生活店舗と昼の人流を主役にする。
- **主役エリア**: ラーメン、居酒屋、本屋、カフェ、ショップ、ゲームセンターなどの店舗列を、連続した商店街ファサードとして作る。各店舗にオーニング、暖簾、看板、提灯を帰属させる。
- **歩行者帯**: アーケード下や店前通りを、車道ではなく歩行者の通行帯として扱う。提灯列、バナーポール、街灯、店を見ている人で横方向の歩行導線を強める。
- **裏側の生活感**: 商店街の背後には住宅や裏路地を少し残す。ゴミ箱、室外機、猫、電柱を置き、商店街の裏にも生活が続いていることを見せる。
- **人流**: 店前の客、アーケードを歩く人、自転車置き場の人、裏路地側の人を分けて配置する。
- **完成密度**: 建物20〜28、家具80〜100、人12〜18、地面パッチ7〜11を目安にする。
- **失敗条件**: 提灯だけ多くて店先の帰属がない。夜街のように見えすぎている。商店街の連続性がない。

### S1-Ch8: 地方の小駅と駅前ロータリー

- **コンセプト**: 駅舎、ホーム、線路、バス停、タクシー乗り場、周辺店舗をまとめ、地方駅前の小さな交通結節点として見せる。
- **主役エリア**: 駅舎を主役にし、ホーム端、線路、新聞スタンド、自販機、駅前の待ち人を駅に帰属させる。駅舎だけでなく、ホームと乗降の気配を必ず作る。
- **駅前ロータリー**: 道路、歩道、バス停、タクシー乗り場を一体で作る。街灯、マンホール、ガードレール、植栽を使って、駅前の交通空間として整理する。
- **周辺店舗と住宅**: 本屋、カフェ、ラーメン店、住宅を駅前に少し置き、看板、自転車、店先ベンチ、待ち人を配置する。駅利用者が立ち寄る店として見せる。
- **人流**: 駅の待ち人、バス待ち、タクシー待ち、店に入る人、歩道を通る人を分けて配置する。
- **完成密度**: 建物20〜28、家具75〜100、人10〜18、地面パッチ8〜12を目安にする。
- **失敗条件**: 駅舎だけでロータリーがない。ホームや線路が駅と対応していない。待ち人が少なく交通結節点に見えない。

### S1-Ch9: 田園農家と畑のある街はずれ

- **コンセプト**: 住宅街から街はずれへ出たことを見せる。密度を下げるのではなく、空白を畑、庭、農道として意味化する。
- **主役エリア**: 農家、温室、納屋、サイロを中心に置き、柵、土パッチ、水たまり、木、農作業中の人をまとめる。農家と畑の関係が一目で分かるようにする。
- **畑と農道**: 土パッチを複数に分け、柵や低木で区画を作る。草地や土を大きな一枚で終わらせず、畑、農道、庭先として読み分けられるようにする。
- **街の終端**: 住宅、ガレージ、小さな店を少し残し、住宅街の終わりを示す。郵便受け、物干し、自転車、植木鉢、猫を帰属させる。
- **人流**: 農作業中の人、農家前の人、店前の人、農道を歩く人を少数配置する。
- **完成密度**: 建物14〜22、家具55〜85、人5〜12、地面パッチ8〜12を目安にする。
- **失敗条件**: 建物が少ないだけのスカスカ地帯になる。草地や土が大きな一枚で終わる。農家と畑の関係がない。

### S1-Ch10: 街はずれの倉庫と消防分署

- **コンセプト**: 住宅街の外れにある倉庫、消防分署、警察詰所として作る。大型工業地帯ではなく、生活圏からインフラ圏へ近づく予兆に留める。
- **主役エリア**: 倉庫列と消防分署・警察詰所を道路に面して置く。倉庫前にはパレット、コンテナ、コーン、バリア、作業人を配置する。消防分署と警察詰所には見張り塔、旗、車止め、待機人を帰属させる。
- **ヤードと裏側**: ガレージや倉庫裏には、積み荷、ゴミ、電柱、電線、室外機、猫を置き、作業場の裏側の密度を作る。
- **生活要素の残し方**: 住宅や小さな生活要素を少し残し、完全な工業地帯ではないことを示す。草地や道路脇には街灯、マンホール、ガードレールを配置する。
- **人流**: 倉庫前の作業人、消防・警察施設の待機人、道路沿いの通行人を分けて配置する。
- **完成密度**: 建物14〜22、家具60〜85、人6〜12、地面パッチ7〜10を目安にする。
- **失敗条件**: 大型港湾ヤードのような雰囲気になる。倉庫前に作業小物がなく、ただ建物が並ぶだけになる。

### S1-Ch11: 踏切と街はずれの終端

- **コンセプト**: Stage 1 の締めとして、住宅街の終端、踏切、濡れた路面、ラーメン店の提灯を使い、街はずれの境界を作る。最終チャンクなので handoff を最優先する。
- **主役エリア**: 踏切を主役にする。線路2本、遮断機、踏切警報機、信号塔、ガードレールをまとめ、線路だけでなく踏切として一目で分かるようにする。
- **街はずれの気配**: ラーメン店や小住宅を道路沿いに置き、暖簾、提灯、店前の人を配置する。住宅街から少し外れた、夕方の境界感を出す。
- **道路沿い**: ガソリンスタンドやコンビニを置き、自販機、看板、街灯、車止めで街はずれ感を作る。濡れた路面、水たまり、街灯を踏切待ちスペースに帰属させる。
- **人流**: 踏切待ちの人、店前の人、コンビニやガソリンスタンド利用者、先へ向かう通行人を分けて配置する。
- **完成密度**: 建物14〜22、家具60〜85、人6〜12、地面パッチ7〜10を目安にする。
- **失敗条件**: 踏切の小物が少なく、線路だけに見える。街はずれへの終端感がない。家具が線路・踏切・道路の関係を隠し、ヒットしたい対象や進行方向が読みにくくなる。
