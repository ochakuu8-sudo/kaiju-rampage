# 怪獣ランページ — 全5ステージ統合ジオラマ設計・実装指示書 完全版

<aside>
🏙️

このページは、`kaiju-rampage` の **全5ステージ 60チャンク**を、Stage 1 と同等の粒度で高品質な見下ろし型ジオラマへ再構成するための **単体完結の設計・実装指示書**です。別ページの設計書を読ませる前提にしません。この1ページだけで、配置思想・実装形式・ステージ別テーマ・チャンク別の配置核・品質保証まで判断できる状態にします。

</aside>

<aside>
🚨

**最重要**: この作業は「小物を増やす」ことではありません。既存または新規のチャンクを、**建物・家具・人・地面・道路が意味を持って配置されたジオラマ**へ再編成する作業です。すべてのオブジェクトに「ここにある理由」を持たせてください。

</aside>

## 0. 到達目標

全5ステージを、以下の **都市縦断ストーリー**として成立させる。

```
Stage 1: 昼の住宅街から街はずれへ
→ Stage 2: 夜の歓楽街・飲食街
→ Stage 3: 都心オフィス・公共中枢
→ Stage 4: 工業港湾・インフラ地帯
→ Stage 5: 城下町・祭礼・最終決戦
```

完成後、画面だけを見て以下が分かること。

- 今いるステージがどんな街か
- 各チャンクが何の場所か
- 人がなぜそこにいるか
- 家具がどの建物・道路・広場・生活痕跡に属しているか
- 地面パッチが場所の輪郭を作っているか
- 前後チャンクとステージが自然につながっているか
- 破壊対象密度と燃料補給源が維持されているか

## 1. 実装前提

### 1.1 対象

```
Repository: https://github.com/ochakuu8-sudo/kaiju-rampage
Branch: main
Primary: src/stages.ts
Reference: src/constants.ts / src/entities.ts / src/scenes.ts / src/game.ts / src/humans.ts
```

### 1.2 raw チャンク形式

全ステージは、可能な限り `raw` チャンクとして手書き構成する。

```tsx
{
  patternId: 's1_raw',
  raw: {
    buildings: [
      _B('house', -120, 40),
    ],
    furniture: [
      _F('bench', 30, 125),
    ],
    humans: [
      _H(30, 125),
    ],
    grounds: [
      _G('concrete', 0, 100, 360, 200),
    ],
    horizontalRoads: [_MID_HR],
    verticalRoads: [..._SPINE_V],
  }
}
```

### 1.3 禁止プロパティ

以下を `src/stages.ts` に入れない。

```tsx
title: '...'
furnitures: [...]
roads: [...]
buildings: [...] // raw の外
furniture: [...] // raw の外
humans: [...]    // raw の外
grounds: [...]   // raw の外
```

### 1.4 道路の変換

```tsx
// 通常
horizontalRoads: [_MID_HR],
verticalRoads: [..._SPINE_V],

// 大きな交差点・ステージ接続・公共軸
horizontalRoads: [_MID_HR, _TOP_HR],
verticalRoads: [..._SPINE_V],
```

### 1.5 型安全

実装前に `BuildingSize` / `FurnitureType` / `GroundType` を確認する。存在しない型は、意味が近い実在型へ置換する。

- 木造住宅: `kominka` / `bungalow` / `house`
    - 古い住宅、郊外住宅として使う
- 現代住宅: `townhouse` / `mansion` / `house`
    - 現代住宅街として使う
- 銭湯: `onsen_inn` + `bathhouse_chimney` + `noren`
    - 建物単体では銭湯に見えないので、家具で補強する
- 夜の雑居ビル: `karaoke` / `game_center` / `izakaya` / `ramen` / `snack` / `club`
    - 実在型を優先する
- 工場: `warehouse` / `factory_stack` / `garage` / `shed`
    - 工業家具で補強する
- 港湾: `warehouse` + `cargo_container` + `pallet_stack` + `forklift`
    - 建物よりヤードで見せる
- 城下町: `kominka` / `machiya` / `kura` / `ryokan` / `chaya` / `pagoda` / `castle`
    - 和風型を優先する

### 1.6 `fallen_leaves`

`fallen_leaves` は `GroundType`。家具として置かない。

```tsx
// NG
_F('fallen_leaves', 0, 100)

// OK
_G('fallen_leaves', 0, 100, 40, 20)
```

## 2. 全ステージ共通ジオラマ設計ルール

### 2.1 2×2 セル

各チャンクは以下の4セルで考える。

```
NW | NE
---+---
SW | SE
```

各チャンクで必ずヒーローセルを定義する。

- 単独: 小公園、銭湯、駅、工場入口、旅館
- 横マージ: 商店街、踏切、駅前、城門、港湾
- 全体: 学校、夜市、コンテナヤード、本丸前広場

### 2.2 4層構造

ヒーローエリアは必ず以下で作る。

1. **焦点**: 一目で場所を表す主役
2. **取り巻き**: 焦点の意味を補強する家具
3. **境界**: 地面・柵・植栽・ガードレールで輪郭を作る
4. **動線**: 人の入口・待機・通行・滞留を示す

### 2.3 密度の作り方

- 建物は意味を作る。空白埋めに使わない
- 家具は密度を作る。ただし必ず帰属先を持たせる
- 地面は輪郭と素材感を作る
- 人は活動を示す。ランダム散布しない

### 2.4 コード核の扱い

各チャンクの「配置の核」は完成配列ではない。実装時は次を行う。

- 既存要素のうちテーマに合うものは残す
- 焦点を邪魔するものは移動する
- 重複・無意味なものは削る
- 足りない取り巻き・境界・動線を追加する
- 最終密度をステージ目安に合わせる

# Stage 1 — 昼の住宅街から街はずれへ

## 連続軸

S1-Ch11 の踏切・濡れた路面・提灯予兆を、S2-Ch0 の夜入口へ引き継ぐ。郊外感は減らし、飲食街へ寄せる。

### Stage 2 → Stage 3

S2-Ch11 で提灯・裏路地を減らし、`street_lamp` / `bollard` / `flag_pole` / `stone_pavement` を増やす。

### Stage 3 → Stage 4

S3-Ch10〜11 で倉庫・搬入口・コンテナを見せ、S4-Ch0 の物流倉庫入口へつなぐ。

### Stage 4 → Stage 5

S4-Ch10〜11 で `stone_pavement`、`pine_tree`、`stone_lantern`、`kominka` を導入し、S5-Ch0 の旧街道入口へつなぐ。

## 全ステージ品質保証

### チャンクごとの合格条件

- [ ]  画面だけで場所名が分かる
- [ ]  ヒーローセルがある
- [ ]  焦点・取り巻き・境界・動線がある
- [ ]  家具が対応する建物や場所に帰属している
- [ ]  人が活動場所にいる
- [ ]  地面が場所の輪郭を作っている
- [ ]  次チャンクまたは次ステージへの handoff がある
- [ ]  破壊対象・人・道路・ボール到達性が維持されている

### ステージごとの合格条件

- [ ]  Stage 1 は生活のある昼の住宅街として読める
- [ ]  Stage 2 は夜の歓楽街として読める
- [ ]  Stage 3 は整然とした都心として読める
- [ ]  Stage 4 は工業港湾として読める
- [ ]  Stage 5 は城下町と最終決戦として読める

### 実装順序

1. union 確認
2. Stage 1 実装 → `npm run build`
3. Stage 2 実装 → `npm run build`
4. Stage 3 実装 → `npm run build`
5. Stage 4 実装 → `npm run build`
6. Stage 5 実装 → `npm run build`
7. GOAL / `castle` / clear 判定確認
8. `npm run dev` で全ステージ通し確認
