# §2 設計の階層

再配置は **3 階層** で進める。上位から下位へ。下位は上位に従属する。

## 階層 1 — ステージ全体のエリア配置

Act の順序と性格、Act 間の連続軸、ランドマークの配置リズム、ヒーロー/アンビエントのジグザグ流れ（§3.6）。**「12 チャンクを 1 本の物語として読ませる」** レベル。

## 階層 2 — エリア内の建物配置（公園モデル）

セル/エリア単位で、焦点建物とその取り巻き家具を **意味のあるミニチュア** として配置する。

<aside>
🎠

**公園モデル（基本の型、Msg 28 由来）**

公園というエリアを作るなら:

- **焦点**: 中央に遊具（複合遊具+ブランコ+砂場+すべり台）をまとめて置く
- **取り巻き**: その周りにベンチが綺麗に並ぶ（保護者の視線方向）
- **境界**: プランター列や生垣で公園の輪郭を作る
- **動線**: 入口に街灯・花壇、歩道が接続する

この「塊としてエリアが読める」感を、保育園・銭湯・校庭・civic plaza・田の字農地など **すべてのランドマーク** に適用する。

</aside>

## 階層 3 — 道路の多様性

全チャンク同じ道路パターン（`_MID_HR` + `_SPINE_V`）ではなく、**Stage 1 は main ブランチ準拠で `_AVE` + `_MID_HR` を基本骨格** とし、Act ごと・チャンクごとに部分幅道路、裏道、アクセス道、踏切を足して性格を変える。横断的要件。

# §3.5 セル分割の基本グリッド

各チャンクは **2×2 の基本セル**に分割する。各セルが公園モデル（焦点+取り巻き+境界+動線）の最小単位となる。

## セル座標

| セル | x 範囲 | y 範囲（ローカル） | facade 線 |
| --- | --- | --- | --- |
| NW | -180 〜 0 | 0 〜 100 | y=22（上段） |
| NE | 0 〜 +180 | 0 〜 100 | y=22（上段） |
| SW | -180 〜 0 | 100 〜 200 | y=118（下段） |
| SE | 0 〜 +180 | 100 〜 200 | y=118（下段） |
- 各セルは **約 180×100** の矩形
- セル境界は論理上、中央 `_AVE`（x=0 の avenue）と水平 `_MID_HR`（y=100）で区切る。旧 `_SPINE_V` の x=-90 / +90 側道は Stage 1 では固定骨格として扱わない
- 上段セル（NW/NE）は facade 線 y=22、下段セル（SW/SE）は y=118

## セルマージ（焦点が大きい場合）

焦点が単一セルに収まらない場合、隣接セルをマージできる。マージ後のセルは 1 つの焦点+取り巻きセットで構成する。

- **NW+NE マージ**: 上段全幅（幅 360、高さ 100）— 商店街アーケード（Ch7）、田園上段（Ch9）
- **SW+SE マージ**: 下段全幅（幅 360、高さ 100）— civic plaza（Ch5）、駅前広場（Ch8）
- **NW+SW マージ**: 左半全高（幅 180、高さ 200）— 校庭や畑など左半分を占める大きな屋外焦点
- **全セルマージ**: 小学校キャンパス（Ch6、校門+校舎+校庭+プール+裏山が全体を占める）

単一セルからスタートし、**焦点が溢れるときだけ**マージする。最初からマージ前提で設計しない。

## セルごとの密度配分

Act の密度下限目安をセル単位に割る。マージセルは割当の合算を**最低ライン**として扱う。

- **建物**: 各セル 1-2 棟を出発点にするが、**上限は設けない**。マージ時や基本形 C クラスターでは、焦点性・読みやすさ・当たり判定が破綻しない範囲で必要な棟数まで増やしてよい。焦点役割のない背景埋め建物は置かない
- **家具**: 各セル 12 個以上を目安にし、**上限は設けない**。ミニチュア感の密度は家具で作る。賑わいセルでは 35 個を超えてもよい
- **人**: 各セル 0 人以上。全セル均等ではなく、賑わいセルに寄せる。上限は設けず、活動が読める範囲で増やしてよい

セル単位で密度を配分することで、「意味のない建物で埋まった区画」を作らず、賑わいは家具の密度で表現する。

# §3.6 ヒーロー / アンビエント階層（チャンク内）

1 チャンクの 4 セル（または全マージ）を全て同等に扱わない。**視覚の山と谷** を作る。

## 2 種類のセル役割

- **ヒーローセル**: チャンク内で **1-2 セル**。§4 の 4 層構造を完備し、公園モデル基本形 A/B/C を適用した「視線が止まる焦点」を持つ
- **アンビエントセル**: 残り 2-3 セル。住宅・路地・空き地・小さな植栽で構成し、焦点を持たない。家具密度は保つ（§3.5 下限以上）。建物は 0-1 棟を基本とするが、密度や読みやすさが不足する場合は、焦点化しない脇役住宅・小屋・店裏要素として追加してよい

## 配置ルール

- **ヒーロー隣接禁止**: ヒーローセル同士を avenue (x=0) / MID_HR (y=100) 両側で隣接させない（2 ヒーロー時は対角配置 = NW+SE または NE+SW）
- **ジグザグ流れ**: 隣接チャンク（北側 / 南側）のヒーロー位置と被らないように、Stage 全体で稲妻型の流れを作る（例: Ch3 NE ヒーロー → Ch4 SW ヒーロー → Ch5 SW+SE マージ → Ch6 全マージ → Ch7 NW+NE マージ）
- **マージチャンクの扱い**: 全マージ（Ch6 小学校）や上段マージ（Ch7 商店街 = NW+NE）/ 下段マージ（Ch5 civic plaza = SW+SE）は「1 大ヒーロー」としてカウント。マージチャンクはアンビエントセルを持たない（例外として OK）
- **Act 境界での連続許容**: Ch5（Act II 末）→ Ch6（Act III 頭）→ Ch7 の 3 連続大ヒーローは **Stage 1 のクライマックス** として許容。ただし、Act 境界 **以外** では大ヒーローチャンクを 3 連続させない（情報過多になる）

## 効果

チャンクをスクロールするとヒーローが交互に現れ、アンビエントセルが呼吸を作る。全セル焦点は「情報過多の地図」になりミニチュア感を失う。**ヒーローが主役、アンビエントが脇役** の役割分担を守る。

# §4 階層 2 — エリア内配置の原則（公園モデル）

## 建物と家具の扱いの違い

<aside>
⚖️

**「詰め込み」の基準は建物数であって家具数ではない**

- **建物は役割のあるものだけ**: 焦点、または焦点クラスター（基本形 C）の一員として意味が明確な建物だけを置く。「このチャンクが寂しいから建物を足す」という判断はしない。棟数の上限は設けず、焦点性・読みやすさ・当たり判定が破綻しない範囲で必要なだけ増やしてよい
- **家具は密度を恐れない**: ミニチュア感は家具の数で作られる。Act 密度目安の家具数は下限として扱い、上限は設けない。賑わいセルでは必要なだけ増やしてよい
- **「スカスカ」はまず家具で埋める**: 建物数の上限は撤廃するが、空間を埋める第一手は家具にする。ベンチ・プランター・自販機・街灯・物干し・エアコン室外機・看板等で密度を上げ、それでも焦点性が必要な場合のみ建物を足す
- **識別性テスト（§4.5 由来）を最優先**: 建物が少なくても取り巻きで建物種が読めれば OK。建物を増やして説明しない
</aside>

## 4 層構造

全セル/エリアは、可能な限り以下の 4 層で構成する。

<aside>
🎯

**焦点 → 取り巻き → 境界 → 動線**

- **焦点 (focal)**: 1-2 棟の建物、または公園/広場/校庭など「無建物エリア」。セルの中心に置く。
- **取り巻き (satellites)**: 焦点に帰属する家具群（看板 / 駐輪 / ベンチ / プランター / 自販機 / 物干し / エアコン室外機 / 煙突 等）。焦点の周囲 10-20 px に集めて「塊」を作る。
- **境界 (boundary)**: セルの輪郭を描く要素（wood_fence / hedge / flower_bed / flower_planter_row / power_pole 等）。セル端 x=±178 付近、y=22 または y=118 の facade 線。
- **動線 (access)**: 歩道の grounds タイル、街灯、マンホール、バス停 等。主要通り（x=0）と中央 avenue（y=100）に沿う。
</aside>

## ミニチュアの 3 つの基本形

- 基本形 A: 公園型（オープンスペース焦点）
    - **焦点**: 遊具/彫像/噴水などの無建物焦点を中央に集約
    - **取り巻き**: 対称または準対称のベンチ + プランター
    - **境界**: hedge / flower_bed / 路面タイル切替
    - **例**: Ch1 児童公園、Ch5 civic plaza、Ch6 校庭
- 基本形 B: 店舗型（単一建物焦点）
    - **焦点**: 1 棟を標準とする商店または公共施設。密度や識別性が不足する場合は、同じ役割の小規模な付属棟を追加してよい
    - **取り巻き**: `a_frame_sign` + `shop_awning` + 駐輪 + パラソル + 自販機（店種による）
    - **境界**: `flower_planter_row` + `potted_plant`

## 左右非対称の原則

- **奇数個で構成する**（benches × 3、planters × 3 等）
- 対の要素を置くときも、わずかに X をずらす（例: statue は x=0 ではなく x=2）
- 家具の「行進」（mailbox が等間隔で一列）は Ch0 で既に 4 個に削減済み。**他チャンクにも同様に適用**
- `_F` 呼び出しは可能な限りクラスタごとにコメント区切り、一列に見えないようにする

## 人流の配置ルール

Act 別の人流グラデ（§3 連続軸サマリ）を、セル内のどこに置くかの原則。

| 状況 | 配置先 |
| --- | --- |
| 店前の客 | 焦点建物の facade 線に接する位置（y=30-35 または y=125-130） |
| ベンチ休憩 | ベンチの隣 ±3 px 以内 |
| 歩行者 | 歩道 grounds 上（x=-65 付近 stone_pavement、または y=100 avenue 脇 y=92/108） |
| 園児・小学生 | 園庭/校庭の dirt パッチ内、または公園焦点の周囲 |
| 野良猫 | 店裏・縁側・畦道・畑・塀際（境界要素の近く、奇数個で不均等） |

- セル内の人配置がそのセルの焦点の「活動」を語るか？（保育園セル → 園児、銭湯セル → 常連老人、駅前セル → 通勤客）

## 取り巻き配置の 3 パターン

§4.5 辞書で選んだ取り巻きを**どう並べるか**の型。1 つの焦点に対し、取り巻きを以下 3 パターンに分類して配置する。

| パターン | 適用する家具 | 配置ルール |
| --- | --- | --- |
| facade 沿い型 | shop_awning / a_frame_sign / noren / chouchin / banner_pole / mailbox | facade 線（上段 y=22 / 下段 y=118）に貼り付け。shop_awning は建物の dx と同じ x、a_frame_sign は店前 ±5 px。左右はゆるく対称（±X シフト 2-5 px） |
| 焦点取り囲み型 | bench / parasol / flower_bed / flower_planter_row / potted_plant / street_lamp / statue / fountain | 焦点から半径 15-30 px の弧上に配置。奇数個（3 / 5）で準対称、対なら X を 2-5 px ずらす。基本形 A（公園型）で主に使用 |
| 歩道沿い型 | 自販機 / bicycle_rack / bicycle_row / traffic_cone / newspaper_stand / bus_stop / taxi_rank_sign | 歩道帯に列で配置。avenue 脇なら y=92 / y=108、x=-65 歩道帯なら x=-60〜-70 の帯。間隔 8-15 px で不均等に |

**適用の優先順**:

1. 署名的取り巻き（`bathhouse_chimney` / `flag_pole` 等、建物アイデンティティを作る物）は **facade 沿い型**
2. 休憩・装飾（`bench` / `parasol` / `flower_bed`）は **焦点取り囲み型**
3. 機能家具（自販機 / 駐輪 / 案内）は **歩道沿い型**

**3 連店舗（基本形 C）の特例**: `shop_awning` は 3 棟連続で 1 つの帯として扱う（個別のファサード付けではなく共通庇）。`parasol` も 3 店舗前で 1 列に並べる。

**「綺麗に並ぶ」 vs 「左右非対称」の両立**:

- パターン単位では綺麗に並べる（bench 3 連は等間隔 + 弧形）
- パターン間で X を 2-5 px ずつずらす（bench 列と parasol 列は同じ x 中心に揃えない）
- 焦点自体も x=0 ではなく x=±2〜5 にずらす

→ ミクロで整列、マクロで不均等、の二層で解決する。

## スケール階層と生活の痕跡

### スケール階層（3 段階の大きさ差）

1 セル内で **大 / 中 / 小** の 3 段階サイズ差を作る。深さと重みを生む。

- **大**: 焦点建物 or オープンスペース焦点（school / grain_silo / 遊具群 / plaza_tile_circle 大パッチ）
- **中**: サテライト構造物（`bathhouse_chimney` / `flag_pole` / `statue` / 大型 `parasol` / `greenhouse` / `fire_watchtower`）
- **小**: 家具（`bench` / `potted_plant` / `a_frame_sign` / `mailbox` / 自販機 / `cat` / `traffic_cone`）

**目安比**: 大 1 : 中 2-3 : 小 10-20。同サイズ要素ばかりが並ぶとミニチュア感が死ぬ。

### 生活の痕跡（"ついさっきまで誰かがいた" 感）

各セルに **1 個以上** 生活感の痕跡要素を置く。機能ではなく雰囲気の要素。

- `laundry_pole` / `laundry_balcony`（物干し。風に揺れる想定はコメントで表現）
- `bicycle_row` / `bicycle_rack` の一台だけ微妙に x/y をずらす（停め方の乱れ）
- `_G('fallen_leaves', x, y, w, h)` 小パッチ（季節感、掃き残し）
- `potted_plant` / `flower_planter_row` の微妙な不揃い（置き主の性格）
- `mailbox` の flag_up ニュアンス（郵便が届いた直後、コメントで意図記述）
- half-open `noren` / `banner` の揺れ（コメントで表現）
- `garbage` / `recycling_bin`（生活ゴミ出し）
- `cat`（既ルール、1 チャンク 1-3 匹、焦点の脇役スポットに）

痕跡がゼロのセルは「ゲームの背景」でしかなく、ミニチュアの愛着は生まれない。

# §5 階層 3 — 道路の多様性（main ブランチ準拠）

<aside>
🛣️

**重要: Stage 1 の道路骨格は旧 `_SPINE_V` ではなく `_AVE` 基準。**

main ブランチの `src/stages.ts` では、Stage 1 raw チャンク用に `_AVE = _VR(0, 0, 200, 'avenue')` が定義され、コメントでも「旧 `_SPINE_V`（x=-90 / 0 / +90）は撤廃。各チャンクが個性的な道路を持つ」と明記されている。`_SPINE_V` は後方互換用として残っているだけなので、Stage 1 の詳細実装では原則 `verticalRoads: [_AVE, ...追加縦道]` を使う。

</aside>

## 現状の問題

v6.1 以前の指示書は、全 12 チャンクが同一の道路構成である前提を置いていた:

- 水平: `_MID_HR`（y=100, -180〜180, 全幅）
- 垂直: `_SPINE_V`（x=-90 street / x=0 avenue / x=+90 street、y=0〜200）

しかし main ブランチの Stage 1 は、すでに **中央 `_AVE` + チャンク固有道路** の方針に移行している。したがって詳細フェーズでは、`_SPINE_V` 固定に戻さず、以下の原則で raw `horizontalRoads` / `verticalRoads` を書く。

## 実装上の道路 API

Stage 1 raw チャンクでは、旧式の `roads: [...]` ではなく、`raw` 内で以下を使う。

```tsx
horizontalRoads: [_MID_HR, ...追加水平道路],
verticalRoads: [_AVE, ...追加縦道路],
```

- `_AVE`: 中央 avenue。`_VR(0, 0, 200, 'avenue')`。Stage 1 の基本縦軸
- `_MID_HR`: 中央クロス街路。`_HR(100, -180, 180)`。基本横軸
- `_TOP_HR`: 上端クロス街路。`_HR(200, -180, 180)`。Act 境界や接続点でのみ使用
- `_HR(dy, xMin, xMax, cls?)`: 部分幅の水平道路。裏道、アクセス道、T字路、搬入路に使う
- `_VR(dx, yMinLocal, yMaxLocal, cls?)`: 部分高さの縦道路。商店街裏、倉庫脇、駅前導線などに使う

## Stage 1 道路の基本方針

- **固定するもの**: 中央 `_AVE`（x=0）と、原則として `_MID_HR`（y=100）
- **固定しないもの**: x=-90 / +90 の縦 street。必要なチャンクだけ、意味のある裏道として `_VR(...)` で個別に足す
- **道路で埋めない**: 道路を増やす目的は密度ではなく、エリアの読みやすさを上げること。スカスカ感は道路ではなく家具・地面パッチ・境界で埋める
- **広場は道路ではなく ground で作る**: 駅前広場、civic plaza、テラス、校庭は `asphalt` / `stone_pavement` / `tile` / `wood_deck` / `dirt` の `_G` で面を作る。道路セグメントを増やして広場を表現しない
- **道路は建物の説明に使う**: コンビニ前の駐車帯、商店裏の搬入路、公共施設前のアクセス道、倉庫脇のサービス道路など、「なぜそこに道があるか」が読めること

## 制約

- Stage 1 では `_SPINE_V` を固定骨格として使わない。`_SPINE_V` は後方互換用で、Stage 1 の新規 raw では原則禁止
- 追加道路は `horizontalRoads` / `verticalRoads` に入れる。`roads: [...]` 形式へ戻さない
- x=-90 / +90 の縦道を追加したい場合も、必ず `_VR(-90, yMin, yMax)` / `_VR(90, yMin, yMax)` として、そのチャンク内の意味をコメントで説明する
- 道路と建物の接触判定は `placeScene` 側では未管理。手動で `xMin` / `xMax` / `dx` / `dy` を調整し、建物が道路に食い込まないようにする
- ground の `asphalt` は道路そのものではなく、駐車場・駅前広場・倉庫前ヤードにも使う。道路として車線を持たせたい場合だけ `_HR` / `_VR` を使う

## 地面パッチの配分原則

Act 基調（§3）と焦点地面（§4 基本形）が衝突しないよう、以下で配分する。

- **非焦点セル**は Act 基調 grounds（例: Act I は `residential_tile`）のみ。パッチを追加しない。
- **焦点セル**のみパッチを持つ:
    - 基本形 A: `tile` / `plaza_tile_circle` / `dirt`（校庭・畑）など焦点専用地面
    - 基本形 B: `wood_deck` 小パッチ（店前 1 棟分、幅 30-50 px）
    - 基本形 C: `wood_deck` 帯（3 棟連続、幅 80-120 px）
- **パッチのサイズ目安**: 焦点（建物 or オープンスペース）の矩形の **1.2 倍前後**を出発点にする。実装後にスカスカに見える場合は広げてよいが、Act 基調が消えないように境界・濃淡・余白で調整する。
- **歩道帯**（x=-65 の `stone_pavement`）は全チャンク共通で、Act 基調とは別扱い。
- **水たまり `puddle_reflection`** は Act IV 以降の湿り気表現として、2-4 箇所に偶発的に置く（焦点セル依存ではない）。
- **見分けテスト**: パッチの切れ目で「どのセルが焦点なのか」が建物を見なくても読めること。

これにより、「Act 基調 → 焦点パッチ → 焦点建物」の 3 層で読ませる。パッチだらけだと基調が死に、焦点が目立たなくなる。

# §6 横断要件

## 座標系

- dx: ワールド X ∈ [-180, +180]
- dy: チャンクローカル ∈ [0, 200]
- チャンク境界は y=0 / y=200
- facade 線は y=22（上段）と y=118（下段）
- 中央 avenue 帯は y=85-115

## ヘルパー（stages.ts 冒頭で定義済み）

- `_B(size, dx, dy)` — 建物
- `_F(type, dx, dy)` — 家具
- `_H(dx, dy)` — 人
- `_G(type, dx, dy, w, h)` — 地面
- `_HR(dy, xMin, xMax, cls?)` — 水平道路
- `_VR(dx, yMinLocal, yMaxLocal, cls?)` — 垂直道路
- `_AVE` — Stage 1 の基本縦軸。中央 avenue（x=0、w=18）。`verticalRoads: [_AVE]` を基本形とする
- `_SPINE_V` — 後方互換用の縦 3 本（x=-90 / 0 / +90）。Stage 1 の新規 raw では原則使わない
- `_MID_HR` — 中央クロス街路（y=100、全幅）。`horizontalRoads: [_MID_HR]` を基本形とする
- `_TOP_HR` — 上端クロス街路（必要なチャンクのみ）
- `horizontalRoads` — raw チャンク内の水平道路配列。`roads` ではなくこちらを使う
- `verticalRoads` — raw チャンク内の縦道路配列。`_AVE` と追加 `_VR` を入れる

## 既存 union の厳守

詳細フェーズで使える型は以下のみ。新規追加は別タスク扱い。

- **FurnitureType**（約 100 種、`entities.ts`）
- **BuildingSize**（約 80 種、`constants.ts`）
- **GroundType**（18 種、`scenes.ts`）

## チャンク境界の連続性

- y=0 / y=200 近傍に対で要素を配置し、隣接チャンクと視覚的に繋げる
- 並木の繋ぎ、歩道の切れ目を作らない
- 電柱は左右端 y=92 / y=195 を対で配置（既存パターン踏襲）

## 精巧さの最低ライン

- 各チャンクの Act ごとの **密度下限目安** を下回らない（**建物数・家具数・人数はいずれも下限**として扱う。上限は設けず、可読性・処理負荷・当たり判定が破綻しない範囲で超過可）
- 各チャンクは **ヒーロー 1-2 セル + アンビエント 2-3 セル** の構成（§3.6）。マージチャンク（Ch5 / Ch6 / Ch7）は「1 大ヒーロー」扱いで例外（アンビエントセルなしで OK）
- 各チャンクに **1 つ以上** 公園モデル 4 層を満たす完成エリアがあること
- 各セルに **生活の痕跡要素 1 個以上**（§4.8: laundry / 傾いた bicycle / fallen_leaves / half-open noren / mailbox flag / trash_bag 等）
- 各セル内で **スケール階層が成立**（§4.8: 大 1 : 中 2-3 : 小 10-20、同サイズ連続禁止）
- 全チャンクで **1 匹以上** の `cat`（象徴的生活感）
- **孤立電柱禁止**: `power_pole` を置いたら `power_line` を 2 本以上接続、または近接電柱と対で配置
- **向きの一貫性**: ベンチ向き / 旗向き / 自販機向き 等が §4.8 の向きルールに沿っていること

## チャンク境界 handoff 記述フォーマット

詳細フェーズで各チャンクの末尾に、**次チャンク冒頭との繋ぎ**を明示する。1 チャンクにつき以下の 4 項目を埋める。

- **並木**: 自チャンク y=180-200 と次チャンク y=0-20 で同じ樹種（sakura_tree / tree / pine_tree）の並木が切れずに連続していること。樹種が切り替わる Ch（Ch5→Ch6、Ch8→Ch9）では**切替地点の x 座標**を明記
- **歩道 / 道路**: x=-65 の stone_pavement 帯（または Act 基調の歩道タイプ）が y=200 ↔ y=0 で連続していること。加えて、Stage 1 では中央 `_AVE`（x=0）が全チャンクで途切れないこと。`_TOP_HR` や部分幅道路は、隣接チャンクと接続する場合だけ handoff に明記する
- **電柱/電線**: y=92 / y=195 の対で、次チャンク冒頭の電柱と電線が繋がること
- **Act 切替の予兆**: Act 末尾チャンク（Ch2 / Ch5 / Ch8 / Ch11）では次 Act の象徴要素を 1-2 個だけ先行配置（例: Act III→IV の Ch8 下段に `pine_tree`）

この 4 項目が埋まらないチャンクは、詳細フェーズの review で戻す。

## セル境界の連続性

チャンク内の **セル境界**（x=0 avenue 両脇、y=100 MID_HR 両側）でも、視覚的不連続を作らない。

- **facade 線の連続**: 上段の NW/NE は facade 線 y=22 で同じ高さに揃える。下段 SW/SE も y=118 で揃える。avenue (x=0) を跨いでも段差ができないこと
- **歩道の連続**: x=-65 の `stone_pavement` 帯は y=0〜200 で切れない。NW→SW の接続で途切れないこと
- **中央 `_AVE` の連続**: x=0 avenue は Stage 1 の基本縦軸として全チャンクで維持する。x=-90 / +90 の旧 spine は固定前提にしない
- **avenue 手前の切れ目禁止**: avenue を跨ぐ歩行者歩道帯（y=92 / y=108）は x=-10〜+10 で切らない
- **焦点の両側バッファ**: 焦点建物は avenue / MID_HR / 追加 `_HR` / 追加 `_VR` から **5 px 以上** 離す（建物が道路に食い込むと不自然）
- **境界フェンス/生垣の通し**: `wood_fence` / `hedge` の列は、セル境界でプツッと終わらせず、avenue 手前で自然にフェードアウト（間隔を徐々に広げる）
- **街路樹の縦方向連続**: 街路樹列（sakura_tree / tree / pine_tree）は NW-SW / NE-SE の縦方向（セルを跨ぐ）でも等間隔を保つ

セルは論理的な区分けであり、視覚的には「なめらかに続く街」として読めること。
